# WebHot 世界状态（World State）与 Timeline 核心战略技术规格书

版本：v1.0

项目名称：WebHot

文档类型：最终核心战略与技术规格说明书

项目核心方向：

```text
Timeline / World State
```

数据底座：

```text
世界变化数据库（Change Database）
```

最终目标：

```text
持续观察世界
```

---

# 1. 项目最终定位

WebHot 最终不应被定义为：

- 热点 API
- 热榜聚合器
- 新闻系统
- 搜索工具
- 舆情工具

这些都只是表层。

WebHot 真正的最终定位应是：

```text
AI Agent 时代的实时世界状态系统
```

其核心目标不是：

```text
告诉 Agent 世界上有什么内容
```

而是：

```text
告诉 Agent 世界正在如何变化
```

这是本项目最重要的战略方向。

---

# 2. 世界变化（World Change）才是核心资产

绝大多数互联网系统：

存储的是：

```text
内容（Content）
```

例如：

- 网页
- 新闻
- 帖子
- 视频
- 推文

但未来真正有价值的是：

```text
变化（Change）
```

例如：

- 什么开始升温
- 什么开始扩散
- 什么开始跨平台传播
- 什么开始金融化
- 什么开始形成生态
- 什么开始形成产业链联动
- 什么开始影响 AI 世界
- 什么开始影响 Agent 世界

因此：

WebHot 的核心数据资产不应是：

```text
新闻数据库
```

而应是：

```text
世界变化数据库（World Change Database）
```

---

# 3. Timeline 将成为系统核心

## 3.1 Timeline 的真正意义

今天绝大多数热点系统：

只能告诉用户：

```text
什么火了
```

但 AI Agent 真正需要的是：

```text
一个事件是如何演化的
```

因此：

Timeline 不是附属功能。

而是：

```text
WebHot 的核心世界模型
```

---

## 3.2 Timeline 的本质

Timeline 本质上是：

```text
世界变化轨迹
```

例如：

```text
MCP 协议
```

Timeline 不是：

```text
一个热点列表
```

而是：

```text
MCP 是如何从一个小众协议
逐步变成 Agent 世界基础设施的
```

这才是真正高价值的数据。

---

# 4. World State（世界状态）系统

## 4.1 为什么 World State 才是终极目标

未来真正高级的 AI Agent：

不应该依赖：

```text
搜索网页
```

而应该依赖：

```text
持续维护世界状态
```

例如：

Agent 应该知道：

```text
AI 世界当前最重要的变化是什么
金融世界当前最重要的变化是什么
Agent 世界正在形成哪些生态
哪些热点正在跨平台扩散
哪些技术正在形成长期趋势
```

因此：

WebHot 的最终形态应是：

```text
World State Engine
```

---

## 4.2 World State 的定义

World State：

并不是静态数据库。

而是：

```text
当前世界状态的动态模型
```

它应该持续维护：

```text
世界正在发生什么
世界正在如何变化
世界哪些方向正在升温
世界哪些方向正在衰退
```

---

# 5. 持续观察世界（Continuous Observation）

## 5.1 这是项目真正的灵魂

WebHot 最重要的目标：

不是搜索。

不是聚合。

不是 API。

而是：

```text
持续观察世界
```

未来的 AI Agent：

真正缺失的能力是：

```text
长期、持续、结构化地观察互联网世界
```

WebHot 正是在解决这个问题。

---

## 5.2 为什么“持续观察”比搜索更重要

搜索只能回答：

```text
现在有什么
```

但无法回答：

```text
它是如何变化的
它为什么变重要
它是如何扩散的
它未来可能走向哪里
```

因此：

WebHot 的价值：

来自：

```text
时间维度
```

---

WebHot 最终不应以：



# 7. 四个核心对象

## 7.1 Timeline

表示：

```text
一个事件是如何演化的
```

例如：

```text
DeepSeek
MCP
Claude Code
Cursor
HBM
GPU
```

Timeline 包含：

- 首次出现
- 热度变化
- 扩散路径
- 平台传播
- 关键节点
- 爆发时刻
- 金融化过程
- Agent 生态形成过程

---

## 7.2 World State

表示：

```text
当前世界状态
```

例如：

```text
当前 AI 世界最重要的话题
当前金融世界最重要的话题
当前 Agent 世界最重要的变化
```

---

## 7.3 Signal

表示：

```text
变化信号
```

例如：

- Explosion Signal
- Trend Signal
- Finance Signal
- AI Signal
- MCP Signal
- Ecosystem Signal

Signal 才是 Agent 真正需要消费的数据。

---

## 7.4 Change

表示：

```text
世界变化记录
```

例如：

```text
什么时候开始升温
什么时候开始跨平台传播
什么时候开始金融化
什么时候开始形成产业链影响
```

---

# 8. 世界变化数据库（Change Database）

## 8.1 Change Database 的真正价值

未来真正值钱的：

不是新闻。

而是：

```text
世界是如何变化的
```

因此：

WebHot 应构建：

```text
世界变化数据库
```

记录：

- 变化轨迹
- 扩散路径
- 趋势演化
- 时间演化
- 平台传播
- 生态形成

---

## 8.2 为什么它会形成护城河

因为：

大部分系统：

没有：

```text
长期历史变化数据
```

而 WebHot：

如果持续运行：

就会拥有：

```text
互联网世界的长期演化轨迹
```

这会形成非常强的数据资产。
